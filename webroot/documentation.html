Presentation

LetDoo PHP est un framework qui fonctionne sous un modèle MVC.
Il est inspiré des fondamentaux du célebre MVC "Cake PHP" et personnalisé pour répondre à des besoins personnels.

LetDoo PHP dispose de quelques librairies PHP et Javascript pour réaliser vos traitements éssentiels en quelques lignes.

LetDoo PHP est optimisé pour une relation avec base MySql. Si vous utilisé un autre SGBD, certaine fonctionnalités seront non fonctionnels.

Dossier Controller

Dans le dossier controller, sera placés les controllers de l'utilisateur.
C'est ici que tous les traitements et initialisations de variables se feront.
Le nom du fichier doit respecter la convention suivante : "nomController.php".
La class hérite du controller principal, le nom de la class doit être le même que celui du fichier mais doit commencer par une majuscule.

class NomController extends Controller{
}

Que faire dans un controller?

Tous les traitements nécéssaires liés à la vue associée:

  1. - Traitement des données utilisateurs
  2. - Lire / Ecrire dans la base de données avec l'utilisation de model
  3. - Gérer les connexions / déconnexions
  4. - etc....

Vous aurez accés aux méthodes du controller principal qui permettent notamment:

  1. De charger un modèle de table pour dialoguer avec la base donnée
  2. D'utiliser les librairies
  3. Définir les variables à transmettre à la vue

Qu'est-ce qu'un model?

C'est une class qui effectueras la connexion avec la base de donnée ciblée.
Elle présente un certain de nombre de méthodes qui permettent d'effectuer des requètes SQL rapidement et simplement.

Dossier Core

Le dossier core n'a pas raison d'être modifié par l'utilisateur (sauf la config). Il contient les routines du MVC.
Vous pouvez en savoir plus sur le contenu dans le chapitre "Class List"

Dossier Model

C'est ici que vous pourrez définir de nouveaux models spécifiques, effectuants des actions particulières par rapport à une table.
La class hérite du model principal, le nom de la class doit être le même que celui du fichier mais doit commencer par une majuscule.

class Table extends Model {

var $table="maTable";
var $primary_key="IdTable";
}
Créer un nouveau model est utile pour réaliser des actions qui sont propres à la table.
Pour réaliser des actions standards, vous pouvez directement appeller le model principal

Dossier View

Dans le dossier view, sera placés les vues de l'utilisateur.
La vue correspont à la page html affichée, où sera envoyé les données du controller
Le nom du fichier doit être le même que le controller auquel il y relié.
Exemple: une vue "index.php" sera reliée au controller "indexController.php"

Un controller n'est pas obligatoirement relié à une vue et peut effectuer des traitements indépendants. Cependant une vue est obligatoirement reliée à un controller.

Que mettre dans une vue?

C'est une page html standart avec peu de code PHP. Le php servira ici notamment à :

  1. - ecrire les variables
  2. - faire quelques conditions / itération

Vous pouvaient aussi appeller des méthodes comme vous le feriez dans le controller. Il faut garder en tête que nous somme dans le controller et $this correspond au controller appellé.

Dossier Webroot

Le dossier webroot est un dossier standart contenant des fichiers css, javascript, image...
Le fichier index est le départ de l'application, c'est est ici que tous sera inclus de facon automatique.

Utilisation

Pour afficher une vue, on appelle le controller correspondant: cela ce fait au niveau de l'URL.
"http://localhost/DossierSiteWeb/controller"
Exemble : "http://localhost/monSite/index" appelle le controller "IndexController.php"

Définissez toujour un controller "IndexController.php", c'est celui appelé par défaut : "http://localhost/monSite/"

Lors de son appel, le controller peut effectuer une action:
"http://localhost/DossierSiteWeb/controller/action"
Exemble : "http://localhost/monSite/index/deconnexion" appelle l'action "deconnexion" dans controller "IndexController.php"

Un controller dispose de trois actions par défaut qui sont appellées si elle sont définies :
"default_action_start" , "default_action_end"
Qui vont s'éxécuter avant et aprés l'action appellée en URL.
Et "default_action" qui s'éxécute uniquement si il n'y a pas d'action appellée.

Une action correspond à une methode définie dans la class du Controller appellé.

class IndexController extends Controller{

public function default_action_start(){
$this->setTitle("Bienvenue sur LetDoo PHP");
}

public function deconnexion{
$this->using("users")->logOut();
}
}
Une méthode peut appellée d'autres méthodes
